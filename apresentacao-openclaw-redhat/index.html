  • Comando: "Move a aplicação para Azure"
  • Navi executou deploy no cluster Azure em segundos

🧠 Como Funcionou

  • Navi criou a apresentação da demo sozinha
  • Kubeconfig de ambos os clusters configurados
  • Cloudflare API integrada para DNS multi-cloud

Demo 2 · Agentes no Kafka

4 AGENTES · KAFKA · CLUSTER K8S AUTÔNOMO

📈

Optimization Agent

Analisa padrões de recursos e sugere otimizações de custo e performance — right-sizing, nodes ociosos, reservas subutilizadas.

🛡️

Security Agent

Monitora vulnerabilidades, acessos suspeitos e compliance — análise contínua de imagem scans, RBAC e network policies.

🔧

Operations Agent

Self-healing de clusters — diagnostica incidentes (CrashLoop, NodeNotReady, OOM) e executa ações corretivas automaticamente.

😈

Troublemaker

Cria problemas controlados no cluster — pods killing, node stress, network chaos — para testar a resiliência dos outros agentes.

📨 Event-Driven

  • Todos os agentes publicam/consomem no Kafka
  • Tópicos: cluster.metrics, cluster.alerts, agent.commands
  • Agentes se coordenam sem orquestrador central
  • Decisões são auditáveis — cada ação vira evento

🎡 A Roleta do Caos

  • Ao final da demo, uma roleta escolhia o problema
  • Pod morto? Node stress? Latência de rede?
  • A roleta disparava o Troublemaker
  • Os outros 3 agentes reagiam em tempo real
  • Cluster se auto-curando — plateia via ao vivo!
12

Fluxo de Operação

📥 Event-Driven

  • Cluster publica eventos no Kafka (métricas, alerts, mudanças de estado)
  • Agentes consomem eventos relevantes por tópico
  • Agente decide se precisa agir ou apenas monitorar
  • Ações são publicadas em tópico de commands
  • Executor no cluster recebe e aplica a ação

🧠 Decisão Inteligente

  • Core Agent usa LLM para analisar contexto completo
  • Histórico de incidentes é consultado na memória
  • Múltiplos agentes podem colaborar via Kafka
  • Fallback humano para decisões críticas
  • Loop de aprendizado contínuo
